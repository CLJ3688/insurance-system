## 核心邏輯說明 (Core Logic)

本專案不僅僅是存入資料，還包含了一個簡單的「保險理賠自動化」邏輯：

1. **唯一案號生成**：使用 `UUID` 技術，確保每一筆申請在系統中都有獨一無二的編號。
2. **自動審核門檻**：在 `Service` 層設定了金額判斷。
   - 若金額 **< 5,000** 元：系統判定為小額理賠，狀態自動設為 `APPROVED`（核准）。
   - 這是為了模擬現實中保險公司為了提高效率，對小額案件進行自動化處理的流程。
3. **資料自動標記**：利用 JPA 的 `@PrePersist` 功能，在資料存入的那一刻自動抓取電腦時間作為「報案日期」。

## 開發實作筆記 (Development Notes)

在開發這個專案的過程中，我遇到了一些具體的挑戰並成功解決了它們：

* **Lombok 環境配置**：
  最初在 Eclipse 中一直出現紅字，顯示找不到 Getter/Setter 方法。後來學習到除了在 `pom.xml` 加入依賴外，Mac 版的 Eclipse 還需要手動將 `lombok.jar` 注入到 `.app` 套件的 `eclipse.ini` 設定檔中。這次經驗讓我更了解開發環境與外掛之間的運作原理。
  
* **API 測試工具應用**：
  第一次使用 **Postman** 進行測試。學會了如何設定 `POST` 請求、如何撰寫正確的 `JSON` 格式，以及如何透過回傳的狀態碼（如 200 OK）來確認後端程式是否運行正常。
